# 📈 股市與財務新聞輿情智慧分析平台

提供台灣特定上市上櫃的公司，透過提供股票基本面與技術面等財務分析資訊，以及籌碼面、新聞情緒等資訊，給予使用者進行股票投資的參考資料，並整合生成式AI，透過綜合股票與財經新聞資訊，提供個股即時的分析。

## 1. 專案目的

在當前股票市場中，一般投資人（散戶）只能透過股市網站所提供的報表資料與新聞資訊進行投資決策，但這往往會存在以下幾點問題：
1. **數據繁多且複雜，缺乏直覺性分析圖表**：若需要透過財報與股市資訊數據進行判斷，需要將資料逐一進行篩選後，才能找出有價值的資訊，並進行投資判斷，這不僅耗時，且容易遺漏重要資訊。
2. **新聞資訊爆炸**：每日產生數百則財經新聞，難以人工逐一閱讀並客觀評估市場情緒，並進行整合分析判斷目前情勢。

**本平台的核心開發目的，就是為了解決上述痛點**。透過整合 **TWSE 官方 API**、**FinMind 金融資料庫**、**集保結算所開放資料**與 **AI 大型語言模型 (FinBERT + Ollama)**，本平台能夠：
* **篩選自選股**：透過股市代號篩選您感興趣的核心個股。
* **季度財務自動避雷**：為個股量身打造「營業現金流、應收帳款、存貨、負債比率、利息保障倍數」五大財務防雷門檻，自動點亮紅/綠燈警示。
* **AI 新聞情緒與因果剖析**：自動抓取最新的新聞，利用 FinBERT 金融情緒分類器與AI分析事件因果分析，並提供即時的投資分析研報。

## 2.分析平台各頁面功能說明

本平台採用 Streamlit 的互動式分析頁面，包含以下四大主要分頁：

### 第一頁：股市看盤 Dashboard
* **個股量價與指標**：繪製股價趨勢圖，疊加20MA（月線）與60MA（季線）移動平均線，並搭配 KD 指標與 RSI 強弱指標。
* **法人籌碼追蹤**：以直方圖直觀呈現「外資」與「投信」每日的買賣超淨額，快速掌握三大法人流向。
* **週股權集中度走勢**：整合集保分散表，動態追蹤「千張大戶持股比率」、「400張大戶持股比率」與「散戶（10張以下）持股比率」的消長與股東總人數變化。

### 第二頁：財務新聞情緒與 AI 綜合分析
* **情緒與股價對照**：將每日新聞的平均情緒分數與收盤價繪製於雙 Y 軸折線圖中，觀察新聞風向與股價的連動性。
* **財經新聞清單**：列出最新新聞標題，並由 FinBERT 中文情緒模型自動標註「正向（綠色）」、「負向（紅色）」與「中立（灰色）」標記。
* **AI 新聞分析**：調用AI模型，自動歸納當前新聞的因果關係，總結市場的核心利多與利空因子。

### 第三頁：財務警訊與閥值設定
* **自訂防雷門檻值**：提供輸入框讓您自訂盈餘品質上限、應收帳款週轉天數、存貨週轉天數、負債比率與利息保障倍數。
* **季度安全燈號**：根據您的設定，自動審查公司最近數季的財務三表（損益表、資產負債表、現金流量表），未達標項目會亮起**紅色警告燈**，合格則亮起**綠色安全燈**。

### 第四頁：股市與新聞綜合分析 AI 報告
* **一鍵生成分析報告**：整合資料庫中最新的每日量價、KD 強弱指標、千張大戶股權比例與季度防雷警告，利用AI進行綜整性分析。
* **自訂問答**：允許使用者輸入特定詢問項目（例如：「評估該公司的庫存風險」），AI 將結合數據給出專業證券投資報告。

## 3. 入門指南 (Getting Started)

### 前置準備 (Prerequisites)
1. **Python 環境**：確保安裝 Python 3.10 或以上版本。
2. **資料庫引擎**：本系統採雙引擎設計。若系統偵測到本機有運行的 PostgreSQL，將優先連線並建表；若無，則會在專案目錄下創建輕量 SQLite 資料庫，無須手動設定。
3. **Ollama 模型**：因 AI 新聞總結與投資報告分析需要透過AI協助生成，請先在本地安裝 [Ollama](https://ollama.com/) 使用本地端模型或申請帳號使用[Ollama雲端模型](https://docs.ollama.com/cloud)。

### 安裝與啟動步驟
```bash
# 1. 複製專案至本地
git clone <本專案GitHub網址>
cd <專案目錄>

# 2. 安裝依賴套件
pip install -r requirements.txt

# 3. 設定環境變數
cp .env
# 編輯 .env 填入您的 FinMind 帳號密碼、本機 PostgreSQL連線設定、Ollama API Key 

# 4. 執行盤後更新管線 (首次執行會自動建立資料庫表格，並抓取歷史資料)
python3 main.py

# 5. 啟動前端 Streamlit 看板網頁
streamlit run streamlit_app.py
```

## 4. 資料來源與爬取流程說明

本平台所使用的所有資料皆來自於公開管道：

1. **公司基本資料**：
   * **來源**：臺灣證券交易所 (TWSE) OpenAPI `/v1/opendata/t187ap03_L`。
   * **處理**：系統在啟動時會自動下載全市場公司基本資料，根據 `STOCK_LIST` 進行篩選。解析民國日曆日期，將產業別數字代號映射至中文名稱。
2. **歷史日K量價與三大法人籌碼**：
   * **來源**：FinMind API (`TaiwanStockPrice`、`TaiwanStockInvestorsBuySell`、`TaiwanStockMarginPurchaseSellBalance`)。
   * **處理**：採用增量抓取機制，查詢資料庫中該股的最新日期，只向下抓取至今日的缺失資料。
3. **週股權集中度資料**：
   * **來源**：台灣集中保管結算所 (TDCC) 開放資料。
   * **處理**：系統會自動繞過並直接下載集保 OpenAPI 的集保分級 CSV，利用 **全域快取 (Memory-Cache)** 機制在一次運行中僅下載一次（40MB），並為 10 檔股票進行記憶體篩選與大戶 (400張/1000張以上) 彙總計算，極大化節省頻寬與效能。
4. **財務基本面 (三表、營收、股利)**：
   * **來源**：FinMind API 財務報告資料集。
   * **處理**：抓取近五年的營收與季度財報，提供給防雷指標計算器進行比對。
5. **財務新聞**：
   * **來源**：Google News RSS。
   * **處理**：系統會自動查詢資料庫中儲存的**公司簡稱** (CompanyNameAbbr)，以簡稱作為檢索詞向 Google News 請求最新的中文財經新聞，並透過 FinBERT-tone-chinese 進行情感極性打分。
