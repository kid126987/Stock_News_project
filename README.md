# 📈 股市與財務新聞輿情智慧分析平台 (sideproV3)

> 歡迎使用 **sideproV3** — 一套專為一般投資人設計的「多維度量化看盤、財務避雷與 AI 新聞輿情」一站式分析平台。

---

## 1. 🎯 專案介紹 (主要建立目的)

在當前的金融市場中，一般零售投資人（散戶）面臨嚴重的「資訊不對稱」與「財務地雷」風險：
1. **地雷股風險**：許多公司帳面獲利亮眼，但實際上應收帳款拖延、庫存積壓严重、負債比率過高，投資人難以從繁複的季度財務三表中一眼看穿警訊。
2. **籌碼與股權混亂**：缺乏直觀的千張大戶與散戶持股流向對照，無法掌握主力資金的真實意圖。
3. **新聞資訊爆炸**：每日產生數百則財經新聞，人工難以逐一閱讀並客觀評估市場情緒。

**本平台的核心開發目的，就是為了解決上述痛點**。透過整合 **TWSE 官方 API**、**FinMind 專業金融資料庫**、**集保結算所開放資料**與 **AI 大型語言模型 (FinBERT + Ollama)**，本平台能夠：
* **自動篩選與追蹤自選股**：透過統一的設定檔一鍵追蹤您感興趣的核心個股。
* **季度財務自動避雷**：為個股量身打造「營業現金流、應收帳款、存貨、負債比率、利息保障倍數」五大財務防雷門檻，自動點亮紅/綠燈警示。
* **AI 新聞情緒與因果剖析**：自動抓取最新的新聞，利用 FinBERT 金融情緒分類器與在地化 Ollama AI 分析師生成事件因果分析，並提供即時的 RAG 投資分析研報。

## 2. 🖥️ 分析平台各頁面功能說明

本平台採用 Streamlit 構建極具科技質感的暗色風（Dark Theme）互動式網頁看板，包含以下四大主要分頁：

### 📊 第一頁：股市看盤 Dashboard
* **個股量價與指標**：繪製精美的日 K 線圖，疊加 20MA（月線）與 60MA（季線）移動平均線，並搭配 KD 指標與 RSI 強弱指標。
* **法人籌碼追蹤**：以直方圖直觀呈現「外資」與「投信」每日的買賣超淨額，快速掌握三大法人流向。
* **週股權集中度走勢**：整合集保分散表，動態追蹤「千張大戶持股比率」、「400張大戶持股比率」與「散戶（10張以下）持股比率」的消長與股東總人數變化。

### 📰 第二頁：財務新聞情緒與 AI 綜合分析
* **情緒與股價對照**：將每日新聞的平均情緒分數與收盤價繪製於雙 Y 軸折線圖中，觀察新聞風向與股價的連動性。
* **精準新聞清單**：列出最新新聞標題，並由 FinBERT 中文情緒模型自動標註「正向（綠色）」、「負向（紅色）」與「中立（灰色）」標記。
* **AI 新聞深度分析**：調用本機 Ollama 模型，自動歸納當前新聞的因果關係，總結市場的核心利多與利空因子。

### 🚨 第三頁：財務警訊與閥值設定
* **自訂防雷門檻值**：提供輸入框讓您自訂盈餘品質上限、應收帳款週轉天數、存貨週轉天數、負債比率與利息保障倍數。
* **季度安全燈號**：根據您的設定，自動審查公司最近數季的財務三表（損益表、資產負債表、現金流量表），未達標項目會亮起**紅色警告燈**，合格則亮起**綠色安全燈**。

### 🤖 第四頁：股市與新聞綜合分析 AI 報告 (RAG)
* **一鍵生成研報**：整合資料庫中最新的每日量價、KD 強弱指標、千張大戶股權比例與季度防雷警告，即時召喚 AI 助理進行 RAG（檢索增強生成）分析。
* **自訂問答**：允許使用者輸入特定詢問項目（例如：「評估該公司的庫存風險」），AI 將結合數據給出專業證券投資報告。

## 3. 🚀 入門指南 (Getting Started)

### 📌 前置準備 (Prerequisites)
1. **Python 環境**：確保安裝 Python 3.10 或以上版本。
2. **資料庫引擎**：本系統採雙引擎設計。若系統偵測到本機有運行的 PostgreSQL，將優先連線並建表；若無，則會自動降級在專案目錄下創建輕量 SQLite 資料庫 (`stock_database.db`)，無須手動設定。
3. **Ollama 模型 (可選)**：若需要啟用 AI 新聞總結與 RAG 投資報告，請先在本地安裝 [Ollama](https://ollama.com/) 並下載 `llama3` 或相關中文模型。

### 🏃‍♂️ 安裝與啟動步驟

```bash
# 1. 複製本專案至本地目錄
cd sideproV3

# 2. 安裝必要的依賴套件
pip install -r requirements.txt

# 3. 複製並設定環境變數
cp .env.example .env
# 編輯 .env 填入您的 FinMind 帳號密碼，以及本機 PostgreSQL (若使用) 連線設定

# 4. 執行盤後更新管線 (首次執行會自動建立資料庫表格，並抓取歷史資料)
python3 main.py

# 5. 啟動前端 Streamlit 看板網頁
streamlit run streamlit_app.py
```

## 📂 專案檔案結構與功能摘要

本專案結構遵循現代軟體工程的最佳實踐，各模組職責清晰（單一責任原則）：

```
sideproV3/
├── main.py                    # 系統入口：依賴檢查、建庫、啟動更新管線與 Streamlit 導引
├── streamlit_app.py           # 看板前端：負責 Tab 頁面佈局、Plotly 圖表渲染與 UI 互動
├── config/
│   └── settings.py            # 全域配置：延遲載入並快取 FinMind API 憑證，管理監控股票清單 (STOCK_LIST)
├── db/
│   ├── engine.py              # 資料庫連線：提供 SQLAlchemy 單例 Engine (相容雙資料庫)
│   ├── schema.sql             # 表格定義：PostgreSQL/SQLite 相容 DDL (包含個股、日K、財報、新聞等 11 張表)
│   └── setup.py               # 資料庫初始化：負責自動建表、 unique 索引檢測與 schema 動態遷移 (Migration)
├── pipeline/
│   ├── data_pipeline.py       # 管線協調器：主導盤後更新流程，依序觸發量價、財報、新聞等更新
│   ├── db_writer.py           # 寫入層：實現白名單過濾防 SQL 注入，及 idempotent_to_sql 冪等防重複寫入
│   ├── indicators.py          # 技術指標計算：計算均線 (MA)、KD 指標 (9,3,3) 與 RSI 強弱指標 (14)
│   ├── sentiment.py           # 新聞情緒分析：封裝 FinBERT 中文情緒模型推理與基於規則的關鍵字回退機制
│   ├── tdcc_scraper.py        # 集保下載器：下載並彙總週股權分散資料，具備 memory-level 快取避免重複下載
│   └── warning_calculator.py  # 財務評估器：比對客製化閾值，計算並 UPSERT 季度防雷指標結果
├── scrapers/
│   └── stock_scraper.py       # 爬蟲包裝：封裝對 FinMind API 與 Google News RSS 的請求
├── services/
│   └── data_service.py        # 資料服務層 (DAO)：為前端提供乾淨的 SQL 查詢封裝與 Ollama AI 提示詞工程
├── tests/
│   └── test_pipeline.py       # 單元測試：使用記憶體 SQLite 對去重寫入、SQL 注入、情緒等功能進行測試
└── requirements.txt           # 套件清單：管理專案所需的第三方庫依賴
```

## 4. 🗄️ 資料來源與爬取流程說明

本平台所使用的所有資料皆來自於公開、合規之管道：

1. **公司基本資料**：
   * **來源**：臺灣證券交易所 (TWSE) OpenAPI `/v1/opendata/t187ap03_L`。
   * **處理**：系統在啟動時會自動下載全市場公司基本資料，根據 `STOCK_LIST` 進行篩選。解析民國日曆日期，並將產業別數字代號映射至中文名稱，存入 `CompanyBaseInfo` 表。
2. **歷史日K量價與三大法人籌碼**：
   * **來源**：FinMind API (`TaiwanStockPrice`、`TaiwanStockInvestorsBuySell`、`TaiwanStockMarginPurchaseSellBalance`)。
   * **處理**：採用增量抓取機制，查詢資料庫中該股的最新日期，只向下抓取至今日的缺失資料。
3. **週股權集中度資料**：
   * **來源**：台灣集中保管結算所 (TDCC) 開放資料。
   * **處理**：因免費版 FinMind 帳戶受限，系統會自動繞過並直接下載集保 OpenAPI 的集保分級 CSV，利用 **全域快取 (Memory-Cache)** 機制在一次運行中僅下載一次（40MB），並為 10 檔股票進行記憶體篩選與大戶 (400張/1000張以上) 彙總計算，極大化節省頻寬與效能。
4. **財務基本面 (三表、營收、股利)**：
   * **來源**：FinMind API 財務報告資料集。
   * **處理**：抓取近五年的營收與季度財報，提供給防雷指標計算器進行比對。
5. **財務新聞**：
   * **來源**：Google News RSS Feed。
   * **處理**：系統會自動查詢資料庫中儲存的**公司簡稱** (CompanyNameAbbr)，以簡稱作為檢索詞向 Google News 請求最新的中文財經新聞，並透過 FinBERT-tone-chinese 進行情感極性打分。
